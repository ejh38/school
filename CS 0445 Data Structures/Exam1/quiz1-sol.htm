a)     Consider the following two interface declarations in Java:

i)      public interface Comparable

{ public int compareTo(Object rhs) }

 

ii)    public interface Comparable<T>

{ public int compareTo(T rhs) }

 

Explain how these declarations differ, the differences in the classes which implement them, and why it is recommended that the second version now be used by Java programmers.  Be specific.

 

Both allow generic behavior.  However, the version in i) requires implementing classes to cast Object to the actual desired type within the method body, whereas the version in ii) utilizes the type parameter T and no casting is required.  Thus, type checking (is the parameter object actually comparable to the current object?) must be done at run-time for version i) whereas this checking can be done at compile-time for version ii).  Compile-time type checking is generally recommended when possible, which is why version ii) is preferred.

 

 

 

 

 

 

 

 

b)     Consider the following Java code:

 

for (int i = 1; i <= N; i++)

     for (int j = 1; j <= i; j++)

          constant_time_operation();

 

State and justify (mathematically) the Big-O runtime in terms on N of the code segment shown.  Be careful!

 

The outer for loop executes N times, iterating on variable i.  The inner for loop executes i times, where i is the current value of the outer loop control variable.  Thus, the constant_time_operation() will execute 1 time when i = 1, 2 times when i = 2, and so on.  This leads to the sum 1 + 2 + 3 + + N = N(N+1)/2.  Ignoring lower order terms and constant multipliers leads to O(N2).
